# Дизайн: Улучшения операционки админ-панели

Дата: 2026-03-08

## Три фичи

### 1. Модель Staff и авторизация

**Новая модель `Staff`:**
- `id`, `name`, `login` (unique), `password` (bcrypt), `role` ("admin" | "staff"), `active` (bool), `createdAt`
- Поле `assignee` в Request → `assigneeId Int?` FK на Staff

**Авторизация:**
- Единая форма входа на /admin: пароль админа (как сейчас) или логин+пароль сотрудника
- Роль определяется автоматически: admin видит всё, staff — урезанный интерфейс

**Права staff:**
- Только страница «Заявки» (без Пользователей, Аналитики, Настроек)
- Только назначенные на него заявки
- Может менять статус и заметки, не может удалять

**Управление:** Новая страница /admin/staff (только admin) — CRUD сотрудников.

### 2. Канбан-доска заявок

- 6 колонок по статусам: Новые → В работе → Ожидание оплаты → На проверке → Выполнено → Отменено
- Drag & drop через `@dnd-kit/core`
- Перетаскивание → PATCH статуса через существующий API
- Карточка: имя, услуга, дата, assignee badge
- Клик раскрывает детали
- Переключатель таблица/канбан уже есть в UI

### 3. Шаблоны ответов

- Новый таб «Шаблоны» в настройках админки
- Для каждого из 6 статусов — редактируемый текст email
- Хранение в `Setting` (key: `template_in_progress`, `template_done` и т.д.)
- При смене статуса — шаблон из БД отправляется клиенту
- Переменные: `{name}`, `{service}`, `{status}`
