# Общее оборудование организации — совместный доступ

**Дата:** 2026-03-07

## Суть

Пользователи могут состоять в нескольких организациях. На странице оборудования — переключатель между "Личное" и каждой организацией. Все участники организации равноправны (CRUD + заявки).

## UI изменения

### Страница оборудования (EquipmentList)

- Вверху — переключатель: `[Личное] [ООО Ромашка] [ООО Василёк]`
- При выборе организации — загружается оборудование с `?organizationId=X`
- Создание оборудования — в выбранном режиме (личное или в орг.)
- Все действия (редактирование, удаление, архивация, заявки) работают одинаково

### Страница организации (/dashboard/organization)

- Остаётся как есть — управление членами, создание организаций
- Без ограничений на количество организаций

## API изменения

- **POST `/api/equipment`** — добавить опциональный `organizationId`; проверять членство
- **PATCH `/api/equipment/[id]`** — разрешить редактирование если пользователь — участник организации оборудования
- **DELETE `/api/equipment/[id]`** — аналогично, проверка через членство
- **POST `/api/equipment/bulk`** — добавить поддержку организационного оборудования
- **POST `/api/equipment/request`** — разрешить заявки из организационного оборудования

## Логика авторизации

Вместо `userId === currentUser` — проверка:
`userId === currentUser ИЛИ equipment.organizationId в списке организаций пользователя`

Вспомогательная функция `canAccessEquipment(userId, equipmentId)` — проверяет владение или членство в организации.

## Что НЕ меняется

- Prisma схема (поле `organizationId` в Equipment уже есть)
- API GET `/api/equipment` (уже поддерживает `?organizationId`)
- Страница организации (создание орг, управление членами)
